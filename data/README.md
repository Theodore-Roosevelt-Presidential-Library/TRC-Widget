## Shape

Terms are stored as **arrays, not objects**. At ~100k terms, repeating the key
names would add several megabytes for zero information. The `fields` property
on each file documents the positional order:

```json
{
  "taxonomy": "dl_creator",
  "label": "Creators",
  "fields": ["id", "name", "slug", "count", "search", "dates"],
  "harvested": "2026-07-18T22:14:03.000Z",
  "total": 41233,
  "terms": [
    [292110, "Roosevelt, Theodore, 1858-1919", "roosevelt-theodore-1858-1919", 38104,
     "roosevelt theodore 1858 1919 | roosevelt theodore | theodore roosevelt",
     ["1858", "1919"]]
  ]
}
```

### The `search` field

This is the reason the widget can do what the live site can't.

TRC stores names in Library of Congress authority format —
`Lodge, Henry Cabot, 1850-1924`. Users type `henry cabot lodge`. On the current
site that matches nothing, while typing `lodge` matches `Blodgett`.

`search` is a normalized string containing every plausible variant of the name,
`|`-separated: authority order, natural order, with and without life dates, with
parenthetical expansions resolved, and each person in a multi-person term indexed
separately. Match a user's typed query against this field, display `name`.

See `scripts/names.mjs` for the derivation and `scripts/names.test.mjs` for the
cases it's guaranteed to handle.

### Ordering

Terms are sorted by `count` descending — most-used first. The widget can take the
first N matches without sorting 100k rows on every keystroke, and the default
suggestions are the ones most likely to be useful.

## Regenerating

```bash
npm run harvest:dry    # probe totals, write nothing
npm run harvest        # full run, resumable
npm run harvest -- --only dl_creator
npm run harvest:fresh  # ignore checkpoints
```

A full harvest is several thousand paginated requests with a deliberate 250ms
pause between each. Expect it to take a while — that's intentional. Progress
checkpoints to `.harvest-cache/` (gitignored), so an interrupted run resumes
where it stopped rather than re-hitting the API from page 1.
